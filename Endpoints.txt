FoodStack API endpoint map (canonical routes)

Auth / Session
- POST /api/login
- POST /api/logout
- GET  /api/session

Users
- GET  /api/users                 (staff only)
- GET  /api/users/{id}            (self or staff)
- POST /api/users                 (public signup -> role forced to customer)
- PUT  /api/users/{id}            (self or staff; role/status only staff)

Categories
- GET  /api/categories
- GET  /api/categories/{id}
- POST /api/categories            (staff only)

Products
- GET  /api/products
- GET  /api/products/{id}
- POST /api/products              (staff only)
- PUT  /api/products/{id}         (staff only)

Ingredients
- GET  /api/ingredients
- GET  /api/ingredients/{id}
- POST /api/ingredients           (staff only)
- PUT  /api/ingredients/{id}      (staff only)

Product Ingredients
- GET  /api/product-ingredients
- GET  /api/product-ingredients/{id}
- GET  /api/product-ingredients/product/{product_id}
- POST /api/product-ingredients   (staff only)
- PUT  /api/product-ingredients/{id} (staff only)

Orders
- GET  /api/orders                (staff only)
- GET  /api/orders/{id}           (owner or staff)
- GET  /api/orders/user/{user_id} (same user or staff)
- POST /api/orders                (authenticated user)
- PUT  /api/orders/{id}           (staff only)

Order Products
- GET  /api/order-products                 (staff only)
- GET  /api/order-products/{id}            (owner or staff)
- GET  /api/order-products/order/{order_id} (owner or staff)
- POST /api/order-products                 (owner of order or staff)
- PUT  /api/order-products/{id}            (staff only)

Order Product Ingredients
- GET  /api/order-product-ingredients
- GET  /api/order-product-ingredients/{id}
- GET  /api/order-product-ingredients/order-product/{order_product_id}
- POST /api/order-product-ingredients
- PUT  /api/order-product-ingredients/{id}

Compatibility notes
- Legacy singular routes are still supported for transition:
  /api/user/{id}, /api/order/{id}, /api/order-product/{id}, etc.
